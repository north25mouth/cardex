0
Node.js をインストール
まだインストールしていない場合は nodejs.org からLTS版をダウンロードしてインストールしてください。
$ node -v
v18 以上が表示されればOKです
1 ZIPを展開してローカルで動かす
1
ZIPを展開する
ダウンロードした cardex.zip をダブルクリックして展開します。
展開すると cardex フォルダが作られます。
展開後のファイル構成
cardex/
├── app/
│   ├── page.tsx          ← メインUI
│   └── api/
│       ├── ocr/          ← OCR + 回数制限
│       └── auth/         ← Googleログイン
├── lib/
│   ├── supabase.ts       ← DB操作
│   └── plans.ts          ← プラン定義
├── supabase/
│   └── schema.sql        ← テーブル定義
├── .env.local.example    ← 環境変数テンプレート
└── README.md
2
ターミナルでフォルダを開く
ターミナル（Mac: Terminal、Windows: PowerShell）を開き、cardexフォルダに移動します。
$ cd cardex
$ npm install
パッケージのインストールに1〜2分かかります
3
環境変数ファイルを作成する
.env.local.example をコピーして .env.local を作ります。この中にAPIキーを書いていきます。
$ cp .env.local.example .env.local
注意： .env.local はGitHubに絶対にアップしないでください。APIキーが漏洩します。
2 Supabase — データベース設定
4
Supabase でプロジェクトを作成
supabase.com にアクセスしてGitHubでログイン。「New project」をクリックしてプロジェクトを作成します。
supabase.com にアクセス
「Start your project」→ GitHubでサインアップ
「New project」→ 名前を cardex に設定
Region を Northeast Asia (Tokyo) に設定
パスワードを設定して「Create new project」
プロジェクトの作成に約1分かかります
5
テーブルを作成する
Supabaseダッシュボードの左メニュー SQL Editor を開き、以下のSQLを貼り付けて「RUN」をクリックします。
SQL
-- usersテーブル（ユーザー管理）
create table users (
  id         uuid primary key default gen_random_uuid(),
  email      text unique not null,
  name       text,
  image      text,
  plan       text not null default 'free'
             check (plan in ('free', 'starter', 'pro')),
  created_at timestamptz not null default now()
);

-- scansテーブル（スキャン回数トラッキング）
create table scans (
  id         uuid primary key default gen_random_uuid(),
  user_id    uuid not null references users(id) on delete cascade,
  scanned_at timestamptz not null default now()
);

-- インデックス（月次集計を高速化）
create index scans_user_month on scans (user_id, scanned_at);

-- RLS（外部からの直接アクセスをブロック）
alter table users enable row level security;
alter table scans enable row level security;
「Success. No rows returned」と表示されればOKです
6
APIキーをコピーする
左メニュー Settings → API を開いて以下の3つをコピーし、.env.local に貼り付けます。
コピーする値.env.local の変数名場所
Project URLNEXT_PUBLIC_SUPABASE_URLSettings → API → Project URL
anon publicNEXT_PUBLIC_SUPABASE_ANON_KEYSettings → API → Project API keys
service_role secretSUPABASE_SERVICE_ROLE_KEY同上（Revealして表示）
3 Google OAuth — ログイン設定
7
Google Cloud で OAuth クライアントを作成
console.cloud.google.com にアクセスして新しいプロジェクトを作成します。
console.cloud.google.com にアクセス
上部のプロジェクト選択 → 「新しいプロジェクト」→ 名前: cardex
左メニュー → 「APIとサービス」→「認証情報」
「認証情報を作成」→「OAuthクライアントID」
アプリケーションの種類: ウェブアプリケーション
承認済みリダイレクトURIに以下を追加：
http://localhost:3000/api/auth/callback/google
本番URLは後でVercelデプロイ後に追加します
8
クライアントID・シークレットをコピー
作成後に表示されるクライアントIDとシークレットを .env.local に貼り付けます。
コピーする値.env.local の変数名
クライアントIDGOOGLE_CLIENT_ID
クライアントシークレットGOOGLE_CLIENT_SECRET
9
NEXTAUTH_SECRET を生成する
ターミナルで以下を実行して出力された文字列を NEXTAUTH_SECRET に貼り付けます。
$ openssl rand -base64 32

ローカルで起動する

5 GitHub にプッシュ
12
.gitignore を確認して GitHub にプッシュ
.env.local がGitHubにアップされないよう、まず .gitignore を作ってからプッシュします。
terminal
# .gitignoreを作成
echo ".env.local\nnode_modules/\n.next/" > .gitignore

# GitHubにプッシュ
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/あなたのユーザー名/cardex.git
git push -u origin main
必ず確認： GitHubのリポジトリに .env.local が含まれていないことを確認してください
6 Vercel にデプロイ
13
Vercel でプロジェクトをインポート
vercel.com にアクセスして GitHubでログイン。「Add New Project」からリポジトリを選択します。
vercel.com にアクセス → GitHubでログイン
「Add New」→「Project」→ cardexリポジトリを選択
Framework Preset: Next.js を確認
「Environment Variables」セクションを開く
14
環境変数をVercelに設定する
.env.local の内容を全てVercelの環境変数に追加します。NEXTAUTH_URL だけは本番URLに変更します。
変数名本番での値
NEXTAUTH_URLhttps://cardex.vercel.app（デプロイ後に確認）
その他全て.env.local と同じ値
全て追加したら「Deploy」ボタンをクリック
15
本番URLをGoogle OAuthに追加
デプロイ完了後に発行された本番URLを Google Cloud Console の承認済みリダイレクトURIに追加します。
https://cardex.vercel.app/api/auth/callback/google
これで完了です。本番URLをブラウザで開いてGoogleログインが動けば成功です。
